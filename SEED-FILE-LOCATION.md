# BusTrack Pro — Seed File Location

## Where is the seed file?

**`prisma/seed.ts`**

This is the main seed script that populates the database with all demo data (accounts, routes, schedules, journeys, traffic alerts, notifications, maintenance records, analytics, and more).

---

## How to reseed the database

```bash
bunx prisma db seed
```

This clears all existing data and inserts fresh demo data. The process takes a few seconds.

---

## How to modify the data

1. Open `prisma/seed.ts` in any text editor
2. Modify the data (names, routes, locations, fares, crew counts, etc.)
3. Run `bunx prisma db seed` to apply changes
4. Changes take effect immediately (restart dev server if needed)

---

## Database file location

**`db/bustrack.db`**

This is the SQLite database file. To completely reset:

```bash
rm -f db/bustrack.db
bun run db:push
bunx prisma db seed
```

---

## Schema file

**`prisma/schema.prisma`**

This defines all database tables and relationships (Profile, CrewProfile, Route, Schedule, Journey, TrafficAlert, Notification, BusMaintenance, RouteAnalytics, SupportTicket, Announcement, etc.).

To push schema changes to the database:

```bash
bun run db:push
```
